Security Features Demonstrated

🛡️

SQL Injection Prevention

All database queries use parameterised placeholders (?). User input is never concatenated into SQL strings — it is always passed as a separate data argument.

Login · Search · All queries
🚫

XSS Prevention

Jinja2 auto-escapes all output by default. Note content is also sanitised with bleach before storage, stripping dangerous HTML tags while allowing safe formatting.

Dashboard · Search results
🔑

CSRF Protection

Every POST form contains a hidden CSRF token generated by Flask-WTF. Requests without a valid token are rejected with HTTP 400.

All forms
✅

Input Validation

WTForms validators enforce field length and format. Custom validators apply allowlist regex to usernames and emails. Password strength is enforced server-side.

Register · New Note
🔒

Password Hashing

Passwords are stored as PBKDF2-SHA256 hashes with a unique random salt per user (via Werkzeug). Plain text is never stored or logged.

Register · Login
📁

File Upload Validation

Uploads are checked against an extension allowlist, validated via magic bytes (not just extension), renamed to a UUID, and size-limited to 2 MB.
